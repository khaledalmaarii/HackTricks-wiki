### Una nota sobre los plugins ‚Äúlist‚Äù vs. ‚Äúscan‚Äù

Volatility tiene dos enfoques principales para los plugins, que a veces se reflejan en sus nombres. Los plugins ‚Äúlist‚Äù intentar√°n navegar a trav√©s de las estructuras del Kernel de Windows para recuperar informaci√≥n como procesos (localizar y recorrer la lista enlazada de estructuras `_EPROCESS` en memoria), manejadores del SO (localizando y listando la tabla de manejadores, desreferenciando cualquier puntero encontrado, etc.). Se comportan m√°s o menos como lo har√≠a la API de Windows si se le solicitara, por ejemplo, listar procesos.

Eso hace que los plugins ‚Äúlist‚Äù sean bastante r√°pidos, pero igual de vulnerables a la manipulaci√≥n por malware que la API de Windows. Por ejemplo, si el malware utiliza DKOM para desvincular un proceso de la lista enlazada `_EPROCESS`, no aparecer√° en el Administrador de tareas ni en el pslist.

Los plugins ‚Äúscan‚Äù, por otro lado, adoptar√°n un enfoque similar a la extracci√≥n de memoria para cosas que podr√≠an tener sentido cuando se desreferencian como estructuras espec√≠ficas. `psscan`, por ejemplo, leer√° la memoria e intentar√° crear objetos `_EPROCESS` a partir de ella (utiliza escaneo de etiquetas de pool, que busca cadenas de 4 bytes que indican la presencia de una estructura de inter√©s). La ventaja es que puede encontrar procesos que han salido, e incluso si el malware manipula la lista enlazada `_EPROCESS`, el plugin a√∫n encontrar√° la estructura en memoria (ya que a√∫n necesita existir para que el proceso se ejecute). La desventaja es que los plugins ‚Äúscan‚Äù son un poco m√°s lentos que los plugins ‚Äúlist‚Äù, y a veces pueden dar falsos positivos (un proceso que sali√≥ hace demasiado tiempo y tuvo partes de su estructura sobrescritas por otras operaciones).

#### **Diferencias entre imageinfo y kdbgscan**

[**Desde aqu√≠**](https://www.andreafortuna.org/2017/06/25/volatility-my-own-cheatsheet-part-1-image-identification/): A diferencia de imageinfo, que simplemente proporciona sugerencias de perfil, **kdbgscan** est√° dise√±ado para identificar positivamente el perfil correcto y la direcci√≥n KDBG correcta (si es que hay m√∫ltiples). Este plugin escanea las firmas KDBGHeader vinculadas a los perfiles de Volatility y aplica verificaciones de sensatez para reducir los falsos positivos. La verbosidad de la salida y el n√∫mero de verificaciones de sensatez que se pueden realizar dependen de si Volatility puede encontrar un DTB, as√≠ que si ya conoces el perfil correcto (o si tienes una sugerencia de perfil de imageinfo), aseg√∫rate de usarlo.

Siempre revisa el **n√∫mero de procesos que kdbgscan ha encontrado**. A veces, imageinfo y kdbgscan pueden encontrar **m√°s de uno** **perfil** adecuado, pero solo el **v√°lido tendr√° alg√∫n proceso relacionado** (Esto se debe a que para extraer procesos se necesita la direcci√≥n KDBG correcta).

